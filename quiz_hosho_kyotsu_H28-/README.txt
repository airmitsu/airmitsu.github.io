【内容】
このフォルダは GitHub Pages 配置用の問題集PWAアプリです。

【配置】
この中身をそのままリポジトリ直下へ置いてください。

【問題データ構成】
problems/
 ├ toi1.txt   ← 問題文
 ├ sen1.txt   ← 選択肢
 ├ kai1.txt   ← 1行目：正解番号、2行目以降：解説
 ├ toi1.png   ← 問題画像（任意）
 ├ kai1.png   ← 解説画像（任意）
 ├ toi2.txt
 ├ sen2.txt
 ├ kai2.txt
 └ ...

【重要ルール】
- ファイルは連番で管理します。
- toi / sen / kai は番号一致必須です。
- 画像は png のみ使用します。
- jpg / jpeg は使いません。
- toi#.txt / sen#.txt / kai#.txt のいずれかが欠けている問題は自動でスキップされます。
- コメント行として使う場合は、行頭に ## を付けてください。

【タイトル】
- アプリタイトルは manifest.webmanifest の name を使用します。
- スマホのアプリ名は short_name を使用します。
- title.txt は使いません。

【読み込み仕様】
- 問題は自動検出です。
- problems-index.json は使いません。
- 問題は全件読み込み後にシャッフルします。
- 読み込みは並列バッチ処理です。
- 画像は表示時に遅延読み込みします。
- 起動後に画像の裏キャッシュを行います。

【今回の実装範囲】
- ホーム画面
- 疑似試験モード
- マスターモード
- 文字サイズ設定
- ローカル保存
- オフライン動作
- 画像ズーム / パン

【注意】
- GitHub Pages では反映に少し時間がかかることがあります。
- PWA更新後、古いキャッシュが残る場合があります。その場合は再読込や再インストールで更新してください。
- icon-192.png / icon-512.png を manifest.webmanifest で使用しています。

【現状の補足】
- 450問規模でも動作する構成です。
- 初回読み込みは問題数と画像数に応じてやや重くなります。
- 画像キャッシュ完了前にオフラインへ切り替えると、一部画像が未取得のままになる場合があります。
